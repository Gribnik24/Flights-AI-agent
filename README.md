# Flights AI Agent

Telegram-бот на базе AI-агента для поиска информации о рейсах, аэропортах и расписаниях. Бот использует LangGraph для построения TAO-цикла (Think, Act, Observe) и обращается к внешним API для получения актуальных данных.

## Возможности

- **Поиск аэропортов** — по названию города или полному названию аэропорта (на русском или английском)
- **Расписание рейсов** — по аэропорту, дате и типу события (вылет/прилёт)
- **Фильтрация рейсов** — по городу отправления/прилёта, времени, авиакомпании
- **Контекстные диалоги** — бот помнит историю переписки и может отвечать на уточняющие вопросы
- **Создание презентаций** — генерация `.pptx` по истории диалога через Presenton API
- **TAO-цикл** — агент мыслит, действует и наблюдает за результатами перед каждым ответом

## Переменные окружения

Все переменные окружения задаются в файле `.env` в корне проекта:

```env
# Telegram Bot
BOT_TOKEN=your_telegram_bot_token_here

# LLM (OpenRouter)
MODEL_NAME=openrouter-model-name
MODEL_API_BASE=https://openrouter.ai/api/v1
MODEL_API_KEY=your_openrouter_api_key

# Yandex Travel API
YANDEX_API_KEY=your_yandex_api_key

# Presenton API (для генерации презентаций)
PRESENTON_API_KEY=your_presenton_api_key

# Путь к папке с промптами из корня проекта (по умолчанию: prompts)
PROMPTS_DIR=prompts

# Системные промты для агента и сервиса создания презентаций
SYSTEM_PROMPT_PATH = agent_system_prompt.txt
PRESENTATION_SYSTEM_PROMPT_PATH = presentation_prompt.txt
```

> **Получение токенов:**
> - `BOT_TOKEN` — через [@BotFather](https://t.me/BotFather) в Telegram
> - `MODEL_API_KEY`, `MODEL_NAME` — на [OpenRouter](https://openrouter.ai/)
> - `YANDEX_API_KEY` — в [Яндекс.Расписания API](https://yandex.ru/dev/rasp)
> - `PRESENTON_API_KEY` — на [Presenton](https://presenton.ai/)

## Использование

| Команда | Описание |
|---------|----------|
| `/start` | Приветственное сообщение с описанием возможностей |
| `/about` | Подробное описание того, что умеет бот |
| `/CreatePresentation N` | Создать презентацию из истории диалога (N слайдов) |
| `/restart` | Очистить историю переписки |

**Примеры запросов:**
- «Найди аэропорты Москвы»
- «Какие рейсы из Шереметьево 15 января?»
- «Покажи прилёты в Домодедо до 12:00»
- «Рейсы из SVO в JFK завтра»

## Архитектура

### TAO-цикл

Агент работает по методологии **Think, Act, Observe**:

1. **THOUGHT** — анализирует запрос и строит план действий
2. **ACTION** — вызывает инструмент для получения данных
3. **OBSERVATION** — анализирует результат и решает, нужно ли повторить цикл

### Инструменты агента

| Инструмент | Описание |
|------------|----------|
| `get_current_time` | Получает текущую дату и время для указанного часового пояса (IANA) |
| `get_airport_data` | Поиск аэропортов по городу или IATA-коду (через SQLite БД) |
| `get_station_timetable` | Получение расписания рейсов через Yandex Travel API с фильтрацией |

### Логи

Бот ведёт два типа логов:

- **Chat logs** (`logs/chat_logs.log`) — события бота: команды, ошибки, запросы к API
- **TAO logs** (`logs/tao_logs.log`) — полный цикл мыслей агента: THOUGHT, ACTION, OBSERVATION, статистика

## Стек технологий

- **Python 3.12+**
- **aiogram 3.28** — фреймворк для Telegram-ботов
- **LangChain + LangGraph** — построение графа агента
- **OpenRouter** — доступ к LLM-моделям
- **SQLite** — локальная база данных аэропортов
- **Яндекс.Расписания API** — актуальное расписание рейсов
- **Presenton API** — генерация PowerPoint-презентаций

## Лицензия
